# AzCord — Self-Hosted Stoat Deployment

A production-ready Docker Compose deployment of [Stoat](https://github.com/stoatchat/stoatchat), an open-source self-hosted chat platform. This setup includes voice/video via LiveKit, optional CloudFlare Tunnel support, and a custom branding layer.

## Architecture

15 containers organized in 3 layers:

**Infrastructure**
- MongoDB — document store
- KeyDB (Redis) — event broker & key-value store
- RabbitMQ — internal message queue
- MinIO — S3-compatible file storage
- Caddy — reverse proxy with automatic HTTPS

**Application**
- `api` — REST API server
- `events` — WebSocket event service
- `autumn` — file server
- `january` — metadata & image proxy
- `gifbox` — Tenor GIF proxy
- `crond` — scheduled task daemon
- `pushd` — push notification daemon
- `voice-ingress` — LiveKit webhook handler
- `livekit` — WebRTC voice/video server

**Frontend**
- `web` — React web application

## Prerequisites

- Docker and Docker Compose v2
- A domain name pointing to your server
- Ports 80, 443, 7881, and 50000–50100/UDP open (or a CloudFlare Tunnel, see below)
- `openssl` installed on the host

## Quick Start

### 1. Clone and configure

```bash
git clone https://github.com/AzLeCurieux/azcord.git
cd azcord
./generate_config.sh your.domain.com
```

The script will:
- Ask whether Stoat sits behind an external reverse proxy (sets Caddy to port 8880 if yes)
- Ask whether to enable camera and screen sharing
- Generate all required cryptographic secrets
- Write `Revolt.toml`, `livekit.yml`, and `.env.web`

> **Important:** Back up the generated `secrets.env` file. Losing it means losing access to all stored files.

### 2. Start the stack

```bash
docker compose up -d
```

Check status:
```bash
docker compose ps
docker compose logs -f
```

### 3. Open in browser

Navigate to `https://your.domain.com` and create your account.

## CloudFlare Tunnel (optional)

Use this if your server is behind NAT or you prefer not to expose ports directly.

### Setup

1. [Create a tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/create-tunnel/) in the CloudFlare Zero Trust dashboard.
2. Download the credentials JSON into `cloudflared/`.
3. Copy the config template:
   ```bash
   cp cloudflared/config.yml.example cloudflared/config.yml
   ```
4. Edit `cloudflared/config.yml` and replace the tunnel ID, credentials path, and hostname.
5. Run the tunnel service alongside the main stack:
   ```bash
   docker compose -f compose.yml -f compose.tunnel.yml up -d
   ```

> The `cloudflared/` directory is gitignored to prevent credential leaks.

## Custom Branding

The `custom-entrypoint.sh` script patches the web app's built assets at container startup, replacing the default app name with a custom one. It runs on every container start so it survives image updates.

To enable it, configure `compose.override.yml`. A template is provided:

```bash
cp compose.override.yml.example compose.override.yml
```

Place custom assets in `custom-assets/`:
- `favicon.ico` — browser tab icon
- `wordmark.svg` — logo shown in the UI

## Configuration Regeneration

To update the domain or regenerate config while keeping existing secrets:

```bash
# Back up current secrets first (they are already in secrets.env if generated by this script)
./generate_config.sh --overwrite your.domain.com
```

## Maintenance

```bash
# Update all images to latest tagged versions
docker compose pull
docker compose up -d

# View logs for a specific service
docker compose logs -f api

# Stop the stack
docker compose down

# Full reset (WARNING: deletes all data)
docker compose down -v
rm -rf data/
```

## Migrations

The `migrations/` directory contains scripts for upgrading between versions:

| Script | Purpose |
|--------|---------|
| `20240929-autumn-rewrite.mjs` | Autumn file server schema migration |
| `20240929-autumn-rewrite---prod-migration.mjs` | Production data migration for Autumn rewrite |
| `20260218-voice-config.sh` | Adds LiveKit voice/video configuration to an existing deployment |

Run a migration:
```bash
node migrations/20240929-autumn-rewrite.mjs
# or
bash migrations/20260218-voice-config.sh your.domain.com
```

## Security Notes

Files excluded from version control (see `.gitignore`):
- `secrets.env` — cryptographic keys
- `Revolt.toml` — generated config containing secrets
- `livekit.yml` — LiveKit keys
- `.env`, `.env.web` — environment variables with domain
- `compose.override.yml` — local overrides
- `cloudflared/*.json` — tunnel credentials
- `data/` — all persistent data volumes
